B — منطق اتصال (Connection Flow)

فلوچارت دقیق از لحظه کلیک تا اتصال نهایی
USER TAPS CONNECT │ ▼ [1] READ SELECTIONS ├── country = AsyncStorage.get('selectedCountry') // e.g. "DE" ├── operator = AsyncStorage.get('selectedOperator') // e.g. "MCI", "MTN" ├── mode = AsyncStorage.get('connectionMode') // "best"|"app"|"auto" └── subType = AsyncStorage.get('subscriptionType') // "public"|"private" │ ▼ [2] FETCH CONFIG LIST IF subType === "private": GET /api/user-sub?code={code} → Base64 decode → parseConfigs() ELSE: GET /api/configs?country={CC}&sort=best&limit=100 → array of raw strings GET /api/sub?country={CC}&limit=100 // fallback │ ▼ [3] PARSE & FILTER forEach(rawConfig): parsed = parseConfig(raw) // vless/vmess/ss/trojan if operator selected: skip if parsed.port not in OPERATOR_PORTS[operator] configArray = parsed.sort(by: quality_score DESC) │ ▼ [4] BATCH TCP TEST (batches of 30, concurrency=15) while(!found && batchIndex < totalBatches): batch = configArray.slice(i*30, (i+1)*30) results = await Promise.allSettled(batch.map(testTCP)) passedConfigs = results.filter(r => r.success) │ ├── IF passedConfigs.length > 0: │ dispatchVotes(passedConfigs) // POST /api/vote batch │ selectedConfig = selectByMode(passedConfigs, mode) │ └── "best" → sort by latency_ms ASC → pick [0] │ └── "app" → sort by quality_score DESC → pick [0] │ └── "auto" → weighted: 0.6*latency + 0.4*quality → pick [0] │ ▼ [5] CONNECTION ESTABLISHED setConnectionState('connected') setBestConfig(selectedConfig) saveToStorage(selectedConfig) showChannelInfo(selectedConfig.telegram_channel) │ └──▶ POST /api/user-sub/report (if private sub) body: { code, volumeMB: 0, activate: true }
🏆

حالت «بهترین»

  1. دریافت تمام کانفیگ‌های کشور انتخابی
  2. تست TCP همه در دسته‌های ۳۰ تایی
  3. مرتب‌سازی بر اساس latency_ms صعودی
  4. اتصال به کمترین latency
  5. ارسال like برای همه موفق‌ها
📱

حالت «برنامه»

  1. دریافت کانفیگ‌های با sort=best از Bot Pro
  2. اولویت‌دهی بر اساس quality_score
  3. تست TCP فقط top-50
  4. اتصال به بالاترین quality موفق
  5. سریع‌ترین حالت (کمترین تست)
⚙️

حالت «اتوماتیک»

  1. دریافت کانفیگ‌های کشور با limit=100
  2. تست TCP همه
  3. محاسبه امتیاز ترکیبی:
  4. score = 0.6/latency + 0.4*quality
  5. اتصال به بالاترین امتیاز

C — State Machine کامل

تمام حالات ممکن اپ و transition های بین آن‌ها
● DISCONNECTED
دکمه قرمز
انتخاب کشور فعال
● FETCHING
دریافت لیست کانفیگ
Spinner + نام کشور
● TESTING [name]
"در حال تست [نام کانفیگ]"
Progress bar + batch info
● CONNECTED
دکمه سبز + glow
نام کانال تلگرام
● ERROR
پیام خطای فارسی
دکمه تلاش مجدد
● NO_CONFIG
هیچ کانفیگ فعالی نیست
پیشنهاد تغییر کشور
type ConnectionState = | 'disconnected' | 'fetching' // downloading config list | { type: 'testing'; configName: string; current: number; total: number } | 'connected' | { type: 'error'; message: string } | 'no_config' // در هنگام تست نمایش دهید: "در حال تست {configName} — {current}/{total}"

D — API Endpoints کامل

تمام endpoint های مورد نیاز بین اپ، FastAPI و Cloudflare Worker
MethodEndpoint (FastAPI)عملکردمنبعوضعیت
GET /api/countries لیست کشورها با تعداد کانفیگ → Worker /api/countries جدید
GET /api/configs کانفیگ‌های raw با فیلتر country/sort → Worker /api/configs جدید
GET /api/sub اشتراک Base64 encoded → Worker /api/sub جدید
POST /api/configs/test-batch تست TCP دسته‌ای داخلی MongoDB موجود
POST /api/vote ارسال batch vote به Worker → Worker /dashboard/api/vote جدید
GET /api/user-sub کانفیگ اشتراک شخصی با کد → Worker /api/user-sub جدید
POST /api/user-sub/report گزارش مصرف حجم → Worker /api/user-sub/report جدید
GET /api/app-update اطلاعات آپدیت اپ → Worker /api/app-update جدید
GET /api/announcements اعلانات فعال → Worker /api/announcements جدید
GET /api/configs/results نتایج تست محلی MongoDB موجود
POST /api/configs/fetch دریافت و parse از URL داخلی بسط
